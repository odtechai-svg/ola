# UI / UX Handoff

## Core interaction principles
- keep sessions short and visually calm
- reduce friction before speech begins
- make progress visible without overwhelming the learner
- communicate why an item is being shown: review, fluency, or new content

## Key screens for the clickable prototype
1. onboarding
2. Today
3. block map
4. Session Player
5. progress
6. settings

## Session Player priorities
- expected sentence and source gloss must be clearly separated
- speaking CTA must be prominent
- feedback must be short, legible, and confidence-building
- progress bar should always show remaining effort
